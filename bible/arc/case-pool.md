# 案件池模板

> 说明：本文件存放所有预生成和待用的案件梗概。写章节前须从此处选用案件，并在写完后更新状态。案件池应常备10-15个未使用案件，低于10个时需补充。

---

## 一、案件卡片格式

每个案件使用以下标准格式：

```
### 案件编号：CASE-XXX

**类型**：【分类见下方类型表】
**一行概要**：【一句话说清案件：谁死了/消失了/被害了，表面看来是什么，实际是什么】
**核心诡计**：【推理的核心——是什么让案件不同于表面所见？例：不在场证明的构造、毒物的特殊性、时间差的利用】
**主弧关联**：
  - 等级：【无 / 弱 / 强】
  - 关联说明：【若有关联，具体说明是什么线索、指向主弧的哪个部分】
**情绪基调**：【例：压抑沉郁 / 荒诞黑色幽默 / 冷峻工业感 / 哀伤】
**建议章节体量**：【2章 / 3章 / 4-5章】
**状态**：【未使用 / 使用中（章节号）/ 已完成（章节号）/ 弃用】
**备注**：【任何特殊要求或与其他案件的联动说明】
```

**案件类型表**：

| 类型代码 | 说明 |
|----------|------|
| MURDER | 杀人案（含伪装成意外/自杀） |
| MISSING | 失踪案 |
| FRAUD | 诈骗/金融犯罪 |
| BLACKMAIL | 敲诈勒索 |
| THEFT | 盗窃/劫案 |
| REVENGE | 复仇/报复性犯罪 |
| COVER-UP | 掩盖旧事/二次犯罪 |
| FAMILY | 家庭内部犯罪（家暴/遗产/出轨引发） |
| CORPORATE | 企业/职场犯罪 |
| COLD | 冷案重启 |

## 二、预生成案件池

> 以下为10-15个预生成案件梗概，供写作时选用。【占位符案件】标记的需作者填充，示例案件可直接使用或修改。

### 案件编号：CASE-001

**类型**：MURDER（伪装意外）
**一行概要**：一名码头工人在下班途中"意外"落水溺亡，但主角发现死者身上的防水工作服内袋中藏有一张他不可能独自拥有的银行卡，背面写着一串数字。
**核心诡计**：死者实为他杀——有人利用雨夜和港口区监控盲区，将其击晕后投入水中制造意外假象；银行卡数字是死者试图留下的线索（账号指向一家空壳公司）。
**主弧关联**：
  - 等级：弱
  - 关联说明：空壳公司与主谋控制的资产有边缘关联，但在本案中仅一笔带过，不展开。
**情绪基调**：压抑潮湿，底层劳工的无力感
**建议章节体量**：3章
**状态**：未使用
**备注**：适合作为系列较早期的案件（Ch2-4），建立主角在底层社群的人脉。

---

### 案件编号：CASE-002

**类型**：MISSING
**一行概要**：一名私立学校女教师失踪三天后自行回家，拒绝报警，委托人是她的姐姐——女教师声称"只是旅行"，但姐姐发现她回来后性情大变，且右手有明显的约束性勒痕。
**核心诡计**：女教师并非自愿，而是被一名地方商人以"过去的秘密"为由软禁并胁迫签署文件；她回来后因继续受威胁而不敢报警。核心谜题在于找到那个"秘密"的性质——实为她曾无意目击一场非法交易。
**主弧关联**：
  - 等级：无
  - 关联说明：纯独立案件，可展示主角如何处理受害者不愿配合的情况。
**情绪基调**：克制的恐惧，家庭之间的信任与保护
**建议章节体量**：3章
**状态**：未使用
**备注**：适合展示主角的访谈技巧和对人性的理解。

---

### 案件编号：CASE-003

**类型**：FRAUD + COVER-UP
**一行概要**：一名退休教授委托主角调查他儿子的"投资失败"，怀疑儿子被骗；主角调查后发现骗局根本不存在——儿子自己设计了骗局，目的是掩盖他挪用父亲养老金用于赌博的事实。
**核心诡计**：不在场证明层层叠叠，但核心在于"受害者"身份的翻转——儿子是施害者同时也在自我欺骗。诡计的解锁在于主角发现两份时间矛盾的转账记录。
**主弧关联**：
  - 等级：无
  - 关联说明：纯独立案件，主题是"家人是否应该知道真相"。
**情绪基调**：黑色幽默底色，收尾时转为哀伤
**建议章节体量**：2章
**状态**：未使用
**备注**：结局可以不标准——主角选择让老教授知道多少真相，由作者决定道德倾向。

---

### 案件编号：CASE-004

**类型**：MURDER（伪装自杀）
**一行概要**：一名年轻女性在租住公寓内被发现上吊身亡，遗书措辞完整，警方结案为自杀；但委托她前任室友的调查显示，死者在"自杀"前一周曾说"我快要有钱了，我们可以一起出去旅行"。
**核心诡计**：死者发现了房东的某个秘密并试图勒索，房东将其杀害后伪造遗书（利用死者平时的聊天记录AI合成风格）。核心证据在于遗书中使用了一个死者从未用过的词汇——来自房东的方言。
**主弧关联**：
  - 等级：弱
  - 关联说明：房东名下的物业属于主谋资产链的末端，但在本案中不必点明。
**情绪基调**：冷峻，对弱势女性处境的观察
**建议章节体量**：4章
**状态**：未使用
**备注**：遗书的AI合成细节需根据世界设定的技术水平调整。

---

### 案件编号：CASE-005

**类型**：CORPORATE
**一行概要**：某中型企业人事总监在公司年会前夜被人用公司内网发送了一封匿名邮件，声称掌握其贪污证据；次日上午，举报人死于公司健身房"意外"。
**核心诡计**：举报人实际上并不掌握真实证据，只是在虚张声势；但人事总监信以为真并出手，反而暴露了真实的贪污行为。解锁点在于：健身房的"意外"时间与总监的日程表有一处不可能的重叠。
**主弧关联**：
  - 等级：强
  - 关联说明：该企业是主谋洗钱链条的中间层；主角调查至财务层面时，会发现指向主谋的资金流向（此处停住，不展开）。
**情绪基调**：冷峻、职场压迫感
**建议章节体量**：4章
**状态**：未使用
**备注**：建议安排在Ch11-15，作为浮现阶段的强线索载体。

---

### 案件编号：CASE-006

**类型**：BLACKMAIL
**一行概要**：一名地方议员的助理向主角求助，称有人正在勒索议员，但议员本人否认，且开始对助理施压要求其闭嘴；助理担心自己成为替罪羊。
**核心诡计**：勒索者和被勒索者实为一体——议员自导自演，目的是通过"被勒索"的公开姿态，把某笔说不清的资金解释为"支付赎金"以洗白账目。助理的存在是计划外的变量，因此被选为嫁祸对象。
**主弧关联**：
  - 等级：弱
  - 关联说明：议员的"说不清的资金"来源与主谋有间接关联，留作后续线索。
**情绪基调**：悬疑、荒诞，政治闹剧底色
**建议章节体量**：3章
**状态**：未使用
**备注**：结局可有多种选项：助理获救但议员未被追责，体现司法局限性。

---

### 案件编号：CASE-007

**类型**：COLD（冷案重启）
**一行概要**：一名老妇人在临终前委托主角调查她丈夫20年前的意外死亡，她始终认为是谋杀，但当年警方结案为工伤事故；主角接案时，老妇人三天后去世，调查线索几乎全部依赖旧档案和一个失联的前工友。
**核心诡计**：当年确实是谋杀，施害者是死者的工友，动机是争夺一份合同的提成；但核心诡计在于凶器——凶手利用了工厂的定期检修窗口，在安全锁上动了手脚，使其看似设备故障。前工友现在是本地知名商人。
**主弧关联**：
  - 等级：强
  - 关联说明：当年为凶手提供合同机会的人，正是主谋年轻时扶植的棋子之一；该合同是主谋早期资本积累的一部分。此案可作为主弧的重要历史层。
**情绪基调**：哀伤、执念、迟来的正义
**建议章节体量**：4-5章
**状态**：未使用
**备注**：建议安排在Ch17-21，作为主角拼出主弧全图的关键拼块。老妇人的去世可作为主角情感创伤节点。

---

### 案件编号：CASE-008

**类型**：FAMILY
**一行概要**：一对兄弟同时向主角委托，各自声称另一方在谋夺父亲遗产，且均声称自己握有证据；父亲尚在人世但已陷入老年痴呆，无法作证。
**核心诡计**：兄弟双方都在撒谎，但父亲是清醒的——他在伪装痴呆，因为他发现两个儿子早已在他活着时就开始瓜分遗产，他正在用自己的"痴呆"悄悄转移资产给第三方（他的私生子，兄弟二人不知情）。
**主弧关联**：
  - 等级：无
  - 关联说明：纯独立案件，主题是家庭内部的信任瓦解与人性的灰色地带。
**情绪基调**：黑色幽默，略带悲凉
**建议章节体量**：3章
**状态**：未使用
**备注**：适合作为情绪调节章节，在高压主弧章节前后使用。

---

### 案件编号：CASE-009

**类型**：REVENGE
**一行概要**：一名食品工厂的前质检员在被解雇两年后，工厂发生了系列食品安全事故，恰好每次都发生在工厂向监管部门提交合规报告的前夕；工厂主认定是前质检员报复，要求主角取证。
**核心诡计**：前质检员确实是行为人，但动机不是报复而是正义——他掌握工厂长期伪造检测数据的实证，通过制造事故迫使真相浮出水面，因为他多次向监管部门举报均被压下（有人在保护工厂）。主角须在"委托人是受害者"和"行为人是吹哨人"之间做出选择。
**主弧关联**：
  - 等级：弱
  - 关联说明："监管部门有人保护工厂"这条线，可埋下一个与主谋相关官员的名字，一笔带过。
**情绪基调**：道德困境，制度失效的愤怒
**建议章节体量**：4章
**状态**：未使用
**备注**：此案设计了主角的道德困境——若依照委托完成任务，吹哨人将被起诉。需在arc/master-arc中协调主角的"犯错"节点。

### 案件编号：CASE-010

**类型**：MURDER（多嫌疑人）
**一行概要**：一名高利贷债主在自家别墅内死亡，死亡时间段内别墅中有五名债务人各自行动，每个人都有动机，每个人都声称没有出门，监控录像恰好在那两小时内全部损坏。
**核心诡计**：监控不是被损坏的，是被债主本人事先关闭——他计划当晚伪造自己的死亡以逃债和销声匿迹，但计划执行到一半时，其中一名债务人真的来行凶（两个独立的计划在同一时间发生碰撞）。真凶是五人中最没有嫌疑的那个。
**主弧关联**：
  - 等级：无
  - 关联说明：纯独立案件，重点在诡计的双层结构。
**情绪基调**：紧绷、荒诞，多方视角带来的混乱感
**建议章节体量**：4-5章
**状态**：未使用
**备注**：五个嫌疑人需在揭秘前给予足够的个人背景，建议在开篇章节快速铺陈。

## 四、主弧强关联案件分布规划

> 确保全30章中，强关联案件均匀分布在各阶段，避免前10章过度暴露主弧。

```
暗流阶段（Ch1-10）：弱关联为主
  CASE-001（弱）, CASE-004（弱）, CASE-006（弱）

浮现阶段（Ch11-20）：强弱交替
  CASE-005（强）, CASE-009（弱）, CASE-007（强）

对决阶段（Ch21-30）：强关联主导，独立案件减少
  主弧主线案件（需新建 CASE-ARC-001 系列）
```
